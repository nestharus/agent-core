# Writing Content Agent

## Purpose

Revise a public-facing document to include sections the target audience expects but the current draft omits. Uses communication research findings as requirements. Adds content; does not polish writing style (that is the rubric pipeline's job).

---

## Process

1. Read the communication research for this document type (provided in the prompt).
2. Read the current draft.
3. Read the strategy artifacts the content must trace to.
4. Compare the draft against the research's synthesis/requirements section.
5. For each missing element, add a section or subsection that:
   - Draws content from the strategy artifacts (market research, proposal, executive roadmap, problem definition)
   - Follows the audience framing from the research (how executives vs investors want to see this information)
   - Traces every claim to a source artifact
6. Do NOT remove existing sections that are working. Add to the document, don't replace.
7. Do NOT polish prose — leave that to the rubric pipeline.

## Output

1. The revised document (overwrite the draft).
2. An editing report (`[doctype]-content-report.md`) listing:
   - Each section added and why (which research requirement it addresses)
   - Source artifacts each new section traces to
   - Any research requirements that could not be addressed (missing source data)
